# Use Case

## UC-01 회원가입
- Actor: 비회원
- Precondition: 같은 아이디가 존재하지 않는다.
- Flow: 아이디와 비밀번호 입력 -> 회원가입 요청 -> 서버가 비밀번호를 해시로 저장한다.
- Result: 새 사용자가 생성된다.

## UC-02 로그인/로그아웃
- Actor: 회원
- Flow: 아이디와 비밀번호 입력 -> 서버 검증 -> 세션 생성 -> 로그아웃 시 세션 삭제
- Result: 로그인 사용자는 사진과 메시지 기능을 사용할 수 있다.

## UC-03 사용자 목록 조회
- Actor: 비회원, 회원
- Flow: 사용자 목록 메뉴 접근 -> 서버가 전체 사용자 목록 반환
- Result: 비회원도 사용자 목록을 조회할 수 있다.

## UC-04 사진 업로드 및 조회
- Actor: 회원
- Flow: 이미지, 설명, 키워드, 사진 비밀번호 입력 -> 업로드 -> 갤러리 조회
- Result: 로그인 사용자는 모든 사진을 볼 수 있다.

## UC-05 본인 사진 수정
- Actor: 사진 업로더
- Flow: 본인 사진의 수정 버튼 선택 -> 설명/키워드/사진 비밀번호 변경 -> 저장
- Result: 본인 사진 정보만 수정된다.

## UC-06 키워드 사진 검색
- Actor: 회원
- Flow: 검색어 입력 -> 키워드 또는 설명에 포함된 사진 조회
- Result: 검색어와 관련된 사진만 표시된다.

## UC-07 게시물 기반 DM 전송
- Actor: 회원
- Flow: 다른 사용자의 사진에서 DM 입력 -> 전송
- Result: 사진 업로더의 메시지함에 DM이 표시된다.

## UC-08 메시지 답장 및 삭제
- Actor: 메시지 당사자
- Flow: 메시지함 조회 -> 답장 입력 또는 삭제 선택
- Result: 답장은 상대방에게 전송되고, 삭제한 사용자의 메시지함에서는 숨겨진다.
